## üìù Contributing to Documentation

### Documentation Standards

**Format:** Markdown (.md files)

**Structure:**
```markdown
# Document Title

> Brief description

## Section 1
Content...

## Section 2
Content...
```

**Code Blocks:**
```markdown
```javascript
// JavaScript code
```

```bash
# Shell commands
```

```json
{
  "json": "example"
}
```
```

### Adding New Documentation

1. Create file in `docs/` directory
2. Add to this INDEX.md
3. Update README.md if needed
4. Follow existing format and style
5. Include table of contents for long docs
6. Add "Last Updated" date at bottom

### Updating Existing Docs

1. Make changes to relevant file
2. Update "Last Updated" date
3. Update INDEX.md if structure changed
4. Commit with descriptive message
