# Phase 6: Shop Floor Execution UX

**Document Version:** 1.0  
**Date:** January 17, 2026  
**Status:** UX Design Specification

---

## 1. EXECUTIVE SUMMARY

This document defines the operator-facing user experience for shop floor execution. The design prioritizes:

- **Tablet-first** interaction (10-12" screens, touch-optimized)
- **Gloves-friendly** large touch targets (minimum 48px)
- **Minimal typing** via barcode scanning, dropdowns, quick-select
- **Visual clarity** in noisy, dusty, variable-lighting environments
- **Fast workflows** for high-volume, fast-turnaround operations

---

## 2. DESIGN PRINCIPLES

### 2.1 Core UX Tenets

| Principle | Implementation |
|-----------|----------------|
| **One-hand operation** | Key actions reachable with thumb on tablet held in one hand |
| **Glove-friendly** | Minimum 48px touch targets, 16px spacing between buttons |
| **Scannable** | Barcode/QR scanning for job/material/operator identification |
| **High contrast** | Dark mode default, high-visibility colors, large fonts |
| **Minimal steps** | Complete common actions in ≤3 taps |
| **Forgiving** | Easy undo, confirmation for destructive actions only |
| **Offline-capable** | Queue actions when network unavailable, sync when restored |

### 2.2 Accessibility Standards

- **Font size:** Minimum 18px body, 24px headings
- **Touch targets:** Minimum 48x48px, recommended 56x56px
- **Color:** Never rely on color alone; use icons + text + color
- **Contrast:** WCAG AA minimum (4.5:1 for text)
- **Feedback:** Haptic + visual + audio confirmation options

### 2.3 Environmental Considerations

| Factor | Design Response |
|--------|-----------------|
| Noise | Visual feedback primary, optional audio alerts |
| Dust/dirt | Sealed tablet cases, large swipe gestures |
| Gloves | Capacitive-compatible gloves required, large targets |
| Variable lighting | Auto-brightness + manual override, high contrast mode |
| Vibration | Secure mounting, large text to reduce blur |

### 3.2 Component Specifications

**Header Bar:**
- Fixed position, always visible
- Operator name + photo (from badge scan login)
- Work center assignment
- Battery/network status
- Settings gear (brightness, sound, logout)

**Status Card (Center):**
- Large visual indicator of current state
- States: READY (green), ACTIVE (blue pulse), PAUSED (yellow), BLOCKED (red)
- Shows active job when running

**Queue List:**
- Scrollable list of assigned jobs
- Color-coded priority indicators (🔴 Critical, 🟡 Hot, 🟢 Standard, ⚪ Economy)
- Due date prominent
- One-tap "START JOB" on first item

**Quick Action Buttons:**
- Large 100x80px touch targets
- SCAN JOB: Opens camera for barcode scan
- REPORT ISSUE: Quick exception entry
- MY STATS: Performance dashboard

**Bottom Navigation:**
- Fixed footer, 5 main sections
- Active state highlighted
- Badge counts for pending items

## 14. EDGE CASE HANDLING

### 14.1 Edge Cases and Solutions

| Edge Case | Detection | UI Response | System Action |
|-----------|-----------|-------------|---------------|
| **Job already started by another operator** | Scan job → server check | "Job J-2450 in progress by Mike S. at LASER-02" | Offer: View only, Take over (supervisor), Report conflict |
| **Material already consumed** | Scan material → inventory check | "This material already used on J-2448" | Block usage, show alternative materials |
| **Work center not assigned to operator** | Scan job → WC check | "You are not assigned to LASER-01 today" | Offer: Request access, Switch to assigned WC |
| **Job on QC hold** | Scan job → status check | "Job awaiting QC approval" | Block start, show QC contact info |
| **Due date already passed** | Start job → date check | "⚠️ This job was due yesterday" | Allow start, flag as late, notify supervisor |
| **No more material available** | Complete job with qty short | "Cannot complete: 5 pieces short, no material" | Escalate to supervisor, create material request |
| **Equipment in maintenance** | Scan job → WC status check | "LASER-01 is under maintenance until 2:00 PM" | Offer alternative work centers or jobs |
| **Power failure during job** | App restart detection | "Job J-2450 was interrupted. Resume?" | Restore last known state, calculate lost time |
| **Multiple operators same job** | Second operator starts | "Join Mike S. on this job?" | Track both operators, split labor time |
| **Wrong operation sequence** | Start op 3 before op 2 | "Previous operation not complete" | Block or allow with supervisor override |
| **Measurement tool disconnected** | Bluetooth disconnect | "Caliper disconnected. Reconnect or enter manually" | Retry connection, fall back to manual entry |
| **Tablet battery critical** | Battery < 10% | "Battery critical. Save progress." | Auto-sync pending transactions, warn user |

| Component | Specification |
|-----------|---------------|
| Tablet | 10.1" - 12.9" screen, rugged case |
| OS | Android 12+ or iPadOS 15+ |
| Processor | Snapdragon 7-series or equivalent |
| RAM | Minimum 4GB |
| Storage | Minimum 64GB |
| Battery | 8000+ mAh for full shift |
| Connectivity | WiFi 6, Bluetooth 5.0 |
| Camera | Rear camera for barcode scanning |
| Optional | Dedicated barcode scanner, NFC for badges |

### 17.2 Performance Targets

| Metric | Target |
|--------|--------|
| App launch | < 3 seconds |
| Screen transition | < 300ms |
| Barcode scan recognition | < 1 second |
| Offline transaction queue | < 100ms to store |
| Sync on reconnect | < 5 seconds for 50 transactions |
| Battery usage | < 15% per hour active use |

### 17.3 Accessibility Features

- High contrast mode (dark/light)
- Large text mode (+25% font size)
- Screen reader compatible
- Colorblind-friendly status indicators
- Haptic feedback for confirmations
- Voice input for descriptions
