## MTR DATA EXTRACTION

### Supported Formats
- PDF (OCR extraction)
- XML (direct parse)
- EDI (856/867 segments)

### Extracted Fields
| Field | Source Location |
|-------|-----------------|
| Heat Number | Header |
| Grade | Specification line |
| Chemistry | Analysis table |
| Mechanical | Test results table |
| Dimensions | Product description |
| Mill Certifications | Footer/stamps |

---

## API ENDPOINTS

```
GET    /api/v1/qc/tests                     # List test results
POST   /api/v1/qc/tests                     # Enter test result
GET    /api/v1/qc/tests/{heat_id}           # Tests by heat
POST   /api/v1/qc/tests/validate            # Validate against spec

GET    /api/v1/qc/holds                     # List holds
POST   /api/v1/qc/holds                     # Create hold
PUT    /api/v1/qc/holds/{id}/disposition    # Resolve hold

POST   /api/v1/qc/mtr/parse                 # Upload & parse MTR
GET    /api/v1/qc/mtr/{heat_id}             # Get MTR data
```
